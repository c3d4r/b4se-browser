# Kolmogorov Complexity

The length of the shortest program (in some fixed universal language) that produces a given output. Also called algorithmic complexity or descriptive complexity. Introduced independently by Solomonoff (1960), Kolmogorov (1965), and Chaitin (1966).

## Core Distinction from Shannon Entropy

Shannon entropy measures the statistical properties of a distribution of outputs. Kolmogorov complexity measures the structure of the generating process.

For most real-world sequences these agree reasonably well. The dramatic exceptions reveal the distinction: pi has maximum Shannon entropy (digits are statistically unpredictable) but trivial Kolmogorov complexity (the entire infinite sequence is generated by a few lines of code).

Shannon entropy asks: how surprising is the next symbol? Kolmogorov complexity asks: how long is the shortest explanation for all the symbols?

## Key Properties

Kolmogorov complexity is formally uncomputable. There is no algorithm that, given an arbitrary string, finds the shortest program producing it. This is proven via a diagonalization argument similar to the halting problem.

Despite uncomputability, the concept is enormously useful as a framework for reasoning about compression, explanation, and understanding.

Kolmogorov complexity is language-invariant up to an additive constant. Choosing a different universal language changes the complexity by at most a fixed amount (the length of the interpreter for one language in the other).

A string is Kolmogorov-random if its complexity equals (approximately) its length. No compression is possible because there is no shorter description than the string itself.

## Relationship to Science

Science, at its core, is the project of finding short programs. A physical law is a compact generator for a large (potentially infinite) set of predictions. F=ma is a short program. Listing every object's acceleration individually is a long description.

Einstein's general relativity (G_uv = 8piT_uv) replaced Newtonian mechanics plus patches (Lorentz contraction, Vulcan hypothesis, etc.) with a shorter program that generated strictly more predictions. This is Kolmogorov compression of physical theory.

## Approximations in Practice

Since Kolmogorov complexity is uncomputable, practical work uses approximations:

- Compression ratio (gzip, bzip2, etc.) as an upper bound on Kolmogorov complexity
- Minimum description length (MDL) for model selection
- Normalized compression distance for sequence similarity
- Lempel-Ziv complexity for online estimation
